## Objectif
Travail sur les fonctionnalités liées à la table Reservation.

Tables concernées :  
- Reservation : id, idClient, nbPassager, dateHeure, idHotel  
- Hotel : id, nom  

BackOffice (BO) : framework maison  
- Formulaire d’insertion de réservation  
- Interface Hotel non développée, insertion directe dans la base  
- Formulaire d’insertion Reservation basé sur le framework interne

FrontOffice (FO) : Spring MVC  
- Liste des réservations par recherche de date  
- Pas de connexion directe à la base  
- Utilisation de l’API exposée par le BackOffice au lieu d’un service interne
